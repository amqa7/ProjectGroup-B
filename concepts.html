Computational Thinking

A problem-solving methodology inspired by how computers process information.

Decomposition

Breaking complex problems into smaller, manageable parts.

Real-world: Planning a trip by separating transportation, lodging, and activities

Programming: Breaking a game into graphics, physics, and scoring systems

Pattern Recognition

Identifying similarities and trends across problems.

Real-world: Recognizing similar steps in different recipes

Programming: Noticing repeated code that can become a function

Abstraction

Focusing on essential details while ignoring irrelevant ones.

Real-world: Using a map instead of every geographical detail

Programming: Using variables instead of hard-coded values

Essential Algorithms

Step-by-step procedures for solving problems.

Searching Algorithms

Linear Search: Checks each item sequentially

Binary Search: Requires sorted data, halves search space each step

Application: Finding names in directories, database queries

Sorting Algorithms

Bubble Sort: Simple but slow (O(n²))

Merge Sort: Efficient (O(n log n)) but uses more memory

Quick Sort: Fast average case, widely used

Graph Algorithms

BFS: Explores all neighbors first (queue-based)

DFS: Goes deep before wide (stack-based)

Dijkstra's: Finds shortest paths in weighted graphs

Fundamental Data Structures

Ways to organize and store data efficiently.

Linear Structures

Arrays: Fixed-size, random access

Linked Lists: Dynamic size, sequential access

Stacks: LIFO (Last-In-First-Out)

Queues: FIFO (First-In-First-Out)

Tree Structures

Binary Trees: Each node has ≤2 children

Binary Search Trees: Ordered for fast search

Heaps: Priority queue implementation

Hash-Based Structures

Hash Tables: Fast key-value lookup

Hash Sets: Stores unique elements

Application: Database indexing, caches
